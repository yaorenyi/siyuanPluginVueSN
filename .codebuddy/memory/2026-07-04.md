## BatchProgressBar — 刷新完成后保持显示 + 手动关闭（14:39）

### 问题
`runBatchWithProgress` 的 `finally` 块调用 `progressEnd()` 立即设 `visible: false`，刷新完成后进度条瞬间消失，用户看不到刚增强的 step 计时日志。

### 方案
区分"自动触发"与"用户主动触发"两类批量操作：前者完成后立即关闭（不挡界面），后者完成后保持显示等用户手动关闭。

### 改动
1. **`types/batchProgress.ts`**：`LoadProgress` 新增 `done?: boolean` 字段
2. **`useBatchProgress.ts`**：新增 `finish()`（停止计时器但保持 visible + 设 done=true）和 `hide()`（设 visible=false）方法
3. **`runBatchWithProgress`**：新增第 5 参数 `options?: { keepVisible?: boolean }`，`finally` 中 `keepVisible ? progressFinish() : progressEnd()`
4. **`BatchProgressBar.vue`**：done 状态显示对勾图标 + "完成" 标签 + 关闭按钮（X），emit `close` 事件；新增 watch(state.done) 完成时自动展开日志
5. **`BatchProgressBar.scss`**：新增 `.gp-batch-progress--done`（进度条半透明）、`.gp-batch-progress-done-icon`、`.gp-batch-progress-close`（hover 变红）
6. **`index.vue`**：
   - 解构增加 `finish: progressFinish, hide: progressHide`
   - `<BatchProgressBar @close="progressHide" />`
   - `handleRefresh` 传 `{ keepVisible: true }`
   - `silentRefreshAll(keepVisible = false)` 加参数，`handleRefreshAll` 调用 `silentRefreshAll(true)`
   - onMounted / watch(activeCategory) / watch(currentView) 不传 keepVisible（默认 false，立即关闭）
