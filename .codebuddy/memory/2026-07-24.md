## gitPush i18n 兜底值清理

- **根因**：`GitPushManager.ts` 中 `const i18n = (this.plugin.i18n as any)?.gitPush || {}`，当思源返回扁平 i18n 时 `.gitPush` 取不到，导致 i18n 始终为 `{}`，所有键 undefined，全靠模板中 `|| '硬编码中文'` 兜底
- **修复**：
  - `GitPushManager.ts`：改为 `i18n = pluginI18n.gitPush || pluginI18n`，兼容嵌套/扁平两种结构
  - `WorkingTreePanel.vue`：全部 17 处 `i18n.xxx || '硬编码中文'` 去掉兜底值，只保留 i18n 引用
- 合并后的 `zh_CN.json` 确认有嵌套结构：`"gitPush": { "workingTreeClean": "工作区干净", ... }`，但 build 产物可能因各种原因未正确加载，兼容写法最稳

## i18n 不生效排查流程（重要）

### 问题现象
i18n 分片文件修改后，思源中显示仍为旧值或 undefined（如 runtime 只有 86 keys，源文件有 257 keys）。

### 排查步骤

1. **确认源文件正确**：`node -e "const j=require('./src/i18n/zh_CN/gitPush.json'); console.log(Object.keys(j.gitPush).length)"`
2. **确认合并文件正确**：源目录 `src/i18n/zh_CN.json` 应为最新合并结果
3. **确认构建产物正确**：找到思源工作区路径（`.env` 文件中的 `VITE_SIYUAN_WORKSPACE_PATH`），检查 `{workspace}/data/plugins/{pluginName}/i18n/zh_CN.json`
4. **手动修复**：如果构建产物是旧的，运行：
   ```
   pnpm i18n:merge
   copy /Y src\i18n\zh_CN.json "{workspace}\data\plugins\{pluginName}\i18n\zh_CN.json"
   copy /Y src\i18n\en_US.json "{workspace}\data\plugins\{pluginName}\i18n\en_US.json"
   ```
   然后重启思源。
