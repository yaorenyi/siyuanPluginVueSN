## gitPush 提交工作流功能（暂存/查看差异/提交/生成提交信息）

**需求**：在推送前增加完整的工作区管理流程 — 查看文件变更、暂存文件、查看差异、生成提交信息并提交。

**实现**（6 个文件变更 + 1 个新文件）：

| 文件 | 变更 |
|------|------|
| `types/storage.ts` | 新增 `FileChange`（path/status/staged/oldPath）和 `WorkingTreeInfo`（branch/files/stagedCount/hasChanges 等）类型 |
| `types/index.ts` | 新增 7 个 Manager 方法：`getWorkingTreeStatus`（git status --porcelain 解析）、`getFileDiff`、`stageFile`/`stageAll`、`unstageFile`/`unstageAll`、`commit`、`generateCommitMessage`（启发式推断 commit type + 文件列表摘要） |
| `composables/useGitPush.ts` | 新增 `workingTrees`/`fileDiffs`/`committing` 3 个 ref + `loadWorkingTree`/`loadFileDiff`/`stageItem`/`stageAllItems`/`unstageItem`/`unstageAllItems`/`doCommit`/`generateCommitMsg` 8 个函数 |
| **`components/WorkingTreePanel.vue`**（新） | 子组件 — 可折叠工作区面板，含文件列表（status 图标/checkbox暂存/diff按钮）、差异查看器（monospace pre）、提交表单（conventional commit 类型按钮 + textarea + 生成/提交按钮）、自动填充 generatedMsg prop |
| `index.vue` | 卡片集成 WorkingTreePanel（remotes 与 push-buttons 之间）；新增 `commitOutputs`/`generatedMsgs` refs + 8 个 handler；`onMounted` 延迟加载工作区状态 |
| `zh_CN.json` / `en_US.json` | 新增 13 个 i18n 键（pendingChanges/workingTreeClean/stageAll/unstageAll/viewDiff 等） |

**工作流**：
1. 面板加载 → 自动检测工作区状态（`git status --porcelain`）
2. 展开工作区 → 显示文件列表（modified/added/deleted/untracked + 颜色标记）
3. 点击 checkbox → 暂存/取消暂存单个文件
4. "暂存全部"/"取消暂存全部" 按钮
5. 每个文件悬停显示差异按钮 → 点击查看 `git diff` 或 `git diff --cached`
6. 暂存后 → 提交表单出现（conventional commit 类型按钮：feat/fix/chore/docs/style/refactor/test）
7. "生成提交信息" → `generateCommitMessage` 基于文件路径启发式推断 type + 文件名列表摘要
8. "提交" → `git commit -m "..."`，提交后自动刷新工作区状态和推送状态

**构建验证**：零 lint 错误，dist/index.js 构建成功。

## 3 个问题修复

### 1. 打开项目文件夹
卡片顶部操作栏新增文件夹图标按钮，点击调用 `shell.openPath()` 打开项目目录。

### 2. 刷新时同步工作区状态
`handleRefresh()` 新增 `await loadWorkingTree(id)` 调用，刷新按钮点击后会同时重检工作区变更，不再需要重启才能看到文件变化。

### 3. execGit 参数含空格导致提交失败
`execGit()` 中 `git ${args.join(" ")}` 对含空格的参数不加引号，导致文件路径和提交信息被拆分为多个参数。修复：对含 `\s"'$\`\\` 的非标志参数自动加双引号包裹。
- 标志参数（`-` 开头）和 `--` 不加引号
- 子命令名（不含特殊字符）不加引号
- 文件路径、提交信息等含空格的自动引号

**修改文件**：`types/index.ts`（execGit）、`index.vue`（handleRefresh + handleOpenPath + 模板按钮）

## 提交信息 AI 生成（接入超级面板大模型 API）

**需求**：生成提交信息不再仅基于文件名启发式推断，而是将完整的 `git diff --cached` 发送给超级面板配置的大模型，让 AI 分析代码变更内容后生成准确的 conventional commit 信息。

**实现**（4 个文件变更）：

| 文件 | 变更 |
|------|------|
| `types/index.ts` | `generateCommitMessage` 重写：获取 `git diff --cached` 全文 → `getApiConfigFromPlugin()` 获取 AI 配置 → 若有 API Key 则 `callAISmart()` 发送 diff + system prompt → AI 返回 commit message；无 Key 或调用失败则降级到 `heuristicCommitMessage()`（原启发式逻辑） |
| `WorkingTreePanel.vue` | 新增 `generating: boolean` prop；生成按钮显示 loading 旋转图标 + "生成中..."文字 |
| `index.vue` | `generatedMsgs` 改为 `generatingMsgs: Record<string, { generating, text }>` 结构；模板传递 `:generating` prop；`handleGenerateMsg` 增加 try/catch + 生成前/后状态切换 |
| `i18n/zh_CN.json` + `en_US.json` | 新增 `generating` 键 |

**AI Prompt 策略**：
```
system: "You are a git commit message generator. Output ONLY conventional commit message."
user:  "根据 git diff，生成 conventional commit 信息（type(scope): description）
       只返回提交信息本身。Diff: {cached_diff truncated to 3000 chars}"
```
参数：temperature=0.3, maxTokens=200，确保输出简洁准确。

**降级策略**：无 API Key / 不在 Electron 环境 / AI 调用失败 → 自动 fallback 到文件路径启发式（feat/fix/chore/docs 等推断）。

**构建验证**：零 lint 错误，dist/index.js 构建成功。

## bugfix: 文件误显示为暂存状态（打勾）且取消暂存后 UI 不更新

**根因**：`execGit()` 中 `stdout.trim()` 对 `git status --porcelain` 输出首行执行前导空格裁剪。`git status --porcelain` 格式为 `XY PATH`（共 2 字符状态码 + 空格 + 路径），未暂存文件的首行 ` M README.md` 被 trim 成 `M README.md`（space 被删），导致解析代码 `statusCode[0] !== " "` 将第一列识别为非空即 `staged=true`，误判为已暂存。

**修复**：`types/index.ts` 中 `execGit` 的 `resolve(stdout.trim())` 改为 `resolve(stdout.replace(/[\r\n]+$/, ""))`，仅移除末尾换行符，保留前导空格/缩进。
