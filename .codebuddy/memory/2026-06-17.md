## Bug 修复：字体族设置不生效

**根因**：动态 CSS (`styles.ts` `applyCodeBlockEnhancedStyles()`) 只设置了 `.code-block .hljs` 的 `font-family`，但思源编辑器的 `.hljs > code` 子元素自带 `font-family`，阻断了继承。

**修复** (`src/features/generalSettings/utils/styles.ts`)：
- 编辑区：`.protyle-wysiwyg .code-block .hljs` → 新增 `.hljs code` 和 `.hljs span` 选择器
- 预览区：`.b3-typography pre` / `pre code` → 新增 `pre span` 选择器
- `font-family` 拆分到独立规则块中（避免和其他属性的 `!important` 混在一起）
