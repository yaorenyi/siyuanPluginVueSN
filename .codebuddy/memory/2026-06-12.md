## gitPush 持久化 bug 修复

**问题**：刷新/重启后项目列表全部消失。

**根因**：`storage.ts` 的 `init()` 方法中 `this.projects.loadOrDefault()` 缺少 `await`（第 49 行），返回 Promise 对象而非数组，`Array.isArray(Promise)` 永远为 false，导致每次 init 都执行 `save(DEFAULT_PROJECTS)`（空数组）覆盖已持久化的项目数据。

**修复**：添加 `await`，同时简化 init() 冗余逻辑——`loadOrDefault()` 已内置「数据缺失回退默认值 + 非数组警告回退」校验，无需 init() 中分两次判断再写回存储。

**修改文件**：`src/features/gitPush/types/storage.ts`（1 个文件，-11 行）

## gitPush 目录选择无反应修复

**问题**：添加项目时点击文件夹图标选择目录后没反应。

**根因**：`selectDirectory()` 使用 `document.createElement("input")` + `webkitdirectory`，但在 Electron 渲染进程中 `files[0].path` 是 Electron 安全策略阻止的非标准属性，选完文件后 `newProjectPath.value` 未赋值。

**修复**：优先使用 Electron 原生 `remote.dialog.showOpenDialog({ properties: ["openDirectory"] })`，兼容新旧 Electron（先用 `@electron/remote`，回退 `electron.remote`），浏览器环境降级到 webkitdirectory。

**修改文件**：`src/features/gitPush/index.vue`（`selectDirectory` 函数重写）

## gitPush 推送判断功能

**需求**：增加判断是否需要推送，避免无变更时的无效推送操作。

**实现**（3 个文件变更 + 新增类型）：

| 文件 | 变更 |
|------|------|
| `types/storage.ts` | 新增 `RemotePushStatus`（ahead/behind/noUpstream/error）和 `PushStatusInfo`（branch/remotes/needsPush）类型 |
| `types/index.ts` | 新增 `checkPushStatus(id)` 方法 — `git rev-list --left-right --count remote/branch...HEAD` 比较 ahead/behind；远程分支不存在时 `rev-list --count HEAD` 获取全部本地提交 |
| `composables/useGitPush.ts` | 新增 `pushStatuses` ref + `loadPushStatus(id)` 函数；`loadProjects()` 自动检测全部项目状态；`pushToBoth`/`pushSingle` 推送后自动刷新状态 |
| `index.vue` | 远程项旁显示状态 badge（↑N / ✓ / ↓N / +N）；按钮在 ahead=0 且非 noUpstream 时禁用；`handleRefresh` 后自动刷新推送状态；新增 3 个辅助函数（statusLabel/statusBadgeClass/needsPushFor） |

**状态 badge 样式**：`gp-ahead`（primary 色）= 需推送，`gp-behind`（warning 色）= 需拉取，`gp-synced`（success 色）= 已同步
