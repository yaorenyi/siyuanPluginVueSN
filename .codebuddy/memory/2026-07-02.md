## gitPush：新增 Fetch 远程状态功能（方案 A+B）

用户需求：跨电脑推送后换机使用，`checkPushStatus()` 只比较本地缓存的远程跟踪分支，behind 始终为 0，`↓N` 不显示。

实施了双重方案：
- **方案 A**：`checkPushStatus()` 增加 `fetchFirst` 选项，手动刷新时先 `git fetch` 更新跟踪分支再比较
- **方案 B**：拉取区新增独立「Fetch」按钮，一键 fetch 所有已配置远程并刷新状态

### 改动文件（5 个）

1. **GitPushManager.ts** — 新增 `fetchRemote()` 私有方法 + `fetchAllForProject()` 公开方法（并行 fetch 各远程）；`checkPushStatus()` opts 增加 `fetchFirst` 参数，传 true 时先调用 `fetchAllForProject`
2. **useGitOps.ts** — `loadPushStatus` 透传 `fetchFirst` 参数；新增 `fetchAllRemotes` action（fetch + 刷新状态）
3. **useGitPush.ts** — 透传 `fetchAllRemotes`
4. **index.vue** — 拉取区新增 Fetch 按钮（cloud-download 图标，带 loading 状态）；`handleRefresh` 传 `fetchFirst: true`；新增 `hasAnyRemote` / `handleFetchAll` 函数
5. **i18n** — zh_CN 和 en_US 各新增 3 个键：`fetch`/`fetchAll`/`fetchHint`/`fetchSuccess`/`fetchError`

## gitPush：修复多本地路径映射导致远程仓库链接丢失

### 问题
当用户在 EditProjectDialog 中更换或新增本地路径（localPaths）时，远程仓库链接（githubUrl 等）丢失。

### 根因（2 处）
1. **GitPushManager.applyRemotesToProject()** — 无条件将所有 `*Remote`/`*Url` 字段重置为 `undefined`，当 `refreshRemotes()` 在跨电脑场景下（新设备 git remote 未配置）被调用时，已有 URL 全部被擦除
2. **index.vue handleEditSaveFromDialog()** — 空字符串 URL 被 `.trim() || undefined` 转为 `undefined` 后直接放入 patch，`Object.assign` 覆盖已有数据

### 修复（2 个文件）
1. **GitPushManager.ts** — `applyRemotesToProject()` 移除预填 `undefined` 的 patch 初始化，仅设置检测到的字段，`Object.keys(patch).length > 0` 时才执行赋值
2. **index.vue** — `handleEditSaveFromDialog()` 构建 patch 后遍历 `["githubUrl","giteeUrl","giteaUrl","cnbUrl"]`，`delete` 掉值为 `undefined` 的字段，避免覆盖已有远程仓库链接

### 验证
- ESLint 0 error
- tsc 仅 3 个预存环境错误（缺少 node/siyuan/vite/client 类型定义），无新增错误

## gitPush：修复编辑弹窗 remote 检测 bug + 移除主路径概念

### 问题
某项目跨电脑使用多路径配置后，编辑弹窗中"Git 远程仓库"列表始终为空，获取不到 `git remote -v` 结果。其他项目正常。

### 根因
`index.vue` `openEditDialog()` 第 1682 行硬编码使用 `project.path` 调用 `detectRemotes`，未使用 `resolveValidPath(project)` 进行多路径解析。主路径在当前设备不存在时 `execGit` 失败。

### 修复（5 个文件）

1. **GitPushManager.ts** — `updateProjectMeta` 的 `Pick` 类型增加 `"path"`
2. **index.vue** — `openEditDialog` 改用 `resolveValidPath(project)` 检测 remote；`editLocalPaths`→`editAllPaths`（统一路径列表 `[project.path, ...localPaths]`）；`handleEditSaveFromDialog` 适配 `allPaths` 拆分为 `patch.path`（第一个）+ `patch.localPaths`（其余）；`handlePickDirForLocalPath` 同步适配
3. **EditProjectDialog.vue** — 移除只读"主路径"区域 + "主路径"徽章，所有路径统一为可编辑列表（输入框+目录选择+删除按钮，至少保留 1 个）；props `localPaths`→`allPaths`；emit `localPaths`→`allPaths`
4. **styles/index.scss** — 移除 `gp-edit-path-primary`、`gp-primary-badge`、`gp-edit-path-hint` 三个废弃 CSS 类
5. **i18n** — 未修改（现有键仍可用，新增无死键）

### 验证

- lint 通过
