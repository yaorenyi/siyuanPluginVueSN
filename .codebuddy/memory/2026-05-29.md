# 2026-05-29

- 修复资源管理器加载全部文件资源报错 `TypeError: path.split is not a function`。原因是 `getUnusedAssets()`、`sql()`、`getDocImageAssets()` 的返回值中可能包含非字符串元素（null/undefined/对象），在 `loadDocNamesForImages` 中调用 `path.split("/")` 时崩溃。在所有数据源头添加了 `typeof p === "string"` 类型守卫过滤，共修改 6 处（loadAllImageAssets×2、loadAllFileAssets×2、loadDocImageAssets×1、loadDocFileAssets×1）。
