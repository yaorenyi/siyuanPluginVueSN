- 新增三种模式切换：全部(all)、当前文档(currentDoc)、指定文档(specifiedDoc)
- "全部"模式：通过 SQL 查询 assets 表获取已引用资源 + getUnusedAssets API 获取未使用资源，合并去重
- "当前文档"模式：使用 getDocImageAssets(docId)（原有行为）
- "指定文档"模式：用户输入文档 ID，使用 getDocImageAssets 查询
- 新增移动/分类功能：每个资源项添加"移动"按钮，展开移动表单
  - 显示当前路径、新路径输入框
  - 快速分类按钮：图片/截图/图标/背景/头像/其他（自动生成 assets/分类名/文件名 路径）
  - 使用 renameAsset API 实现移动（自动更新所有引用）
- 新增 i18n 字段：allAssets, currentDoc, specifiedDoc, docIdPlaceholder, docIdRequired, assetCount, moveAsset, moveSuccess, moveFailed, movePathPlaceholder, currentPath, category, confirmMove, cancel, loadFailed
- 样式：新增 .rm-filter-bar, .rm-asset-count, .rm-move-form 组件样式
- asset-item 添加 flex-wrap: wrap 支持移动表单换行

## 修复 renameAsset 静默失败问题

- 根因：`request()` 函数在 API 返回 `code !== 0` 时返回 `null` 而非抛出异常。对于 `renameAsset` 这种成功也返回 `null` 的 void 接口，调用方无法区分成功/失败
- 修复：`api.ts` 中 `renameAsset` 直接用 `fetchSyncPost` 并检查 `response.code`，非 0 时抛出 `Error(response.msg)`
- 两个调用点 `handleMoveAsset` 和 `handleRenameAsset` 都有 try-catch，能正确捕获并显示错误
