- 修复 resourceManager 移动图片后文件变成 52 字节的 bug（多轮迭代）。
  - 根因：旧代码用 `insertLocalAssets` 复制后 `Object.keys(succMap)[0]` 取的是绝对路径 key 而非资产路径 value，导致 `renameAsset` 收到错误 oldPath → 52 字节损坏。
  - 关键发现：`renameAsset` API 返回成功但**不做物理移动也不更新文档内容**，只更新内部资产表。
  - 修复方案：`renameFile("/data/old", "/data/new")` 物理移动文件 + SQL+`updateBlock` 手动更新引用 + `fullReindexAssetContent` 重建索引。路径格式必须 `/data/assets/...`。
  - 同步修复了 `handleDeleteAsset`（被引用资源用 `removeFile` 替代无效的 `removeUnusedAsset`）。
  - 提取 `loadDocImageAssets(docId)` 消除 currentDoc/specifiedDoc 模式的重复代码。
  - 删除了重命名资源功能（renameAsset tab、renameForm/renameResult、handleRenameAsset 函数），因为移动功能已覆盖重命名场景。
  - 删除了插入本地资源和解析路径功能。
  - 删除了图片资源 tab 下的删除按钮和 `handleDeleteAsset` 函数。
  - Tab 选中色改为 `var(--b3-theme-primary)` 适配暗色主题。
  - 新增"文件资源"tab，展示 zip/txt 等非图片资源，和图片资源共用 UI（模式切换栏+列表），通过 `IMAGE_EXT` 常量区分，`currentAssetList` computed 动态切数据源。
