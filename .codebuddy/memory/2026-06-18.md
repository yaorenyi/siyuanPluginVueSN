## 修复 gitPush IDE 启动卡顿问题

### 根因
`useIdeManagement.ts` 中 `launchIde()` 使用 `cp.exec`（Windows 默认 `shell: true`），导致 IDE 通过 `cmd.exe` 间接启动，成为思源 Electron 的孙子进程，继承了 Electron 资源（文件句柄、GPU 上下文等）导致卡顿。手动直接打开 IDE 是独立顶级进程，无此问题。

### 修复
仅改 1 个文件 `src/features/gitPush/composables/useIdeManagement.ts`，4 处修改：

1. **`launchIde()`**：`cp.exec` → `cp.spawn(detached: true, stdio: 'ignore')`，使 IDE 脱离思源进程树
2. **`isCmdAvailable()`**：同步改为 `cp.spawn` + `error/close` 事件判断
3. **`findSlnFile()`**：`readdirSync` → `fs.promises.readdir` 异步，消除主线程阻塞
4. **`handleOpenCustomIde()`**：`findSlnFile` 调用加 `await`

构建验证通过，零 lint 错误。
