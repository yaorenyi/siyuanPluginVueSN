# 加密功能 UI 优化说明

## 优化概述

对解密功能的用户界面进行了重构，使其更加直观和用户友好。

## 优化内容

### 1. 两步式解密流程

**之前**：输入密码后直接替换文本，无法预览

**现在**：
- **第一步**：输入密码并解密
- **第二步**：显示解密结果，用户可选择操作

### 2. 解密结果界面

解密成功后，对话框会转换为结果展示界面，包含：

#### 界面元素
- ✅ 成功提示标题
- 📝 解密内容预览区（可滚动的文本框）
- 🎯 三个操作按钮：
  - **关闭**：不做任何修改，关闭对话框
  - **复制内容**：将解密内容复制到剪贴板
  - **替换加密文本**：用解密内容替换原文

#### 交互优化
- 解密内容自动选中，方便快速复制
- 复制按钮点击后显示"已复制"反馈
- 替换操作需要用户主动点击，避免误操作
- 所有操作都有明确的提示信息

### 3. 错误处理

- 密码为空时显示提示
- 密码错误时显示错误信息
- 解密按钮在处理时显示"解密中..."状态

### 4. 视觉设计

- 使用思源主题变量，适配深色/浅色模式
- 圆角设计，现代化界面
- 清晰的视觉层次
- 合理的间距和布局

## 用户体验改进

### 改进点 1：安全性提升
用户可以先预览解密内容，确认无误后再决定是否替换原文，避免误操作。

### 改进点 2：灵活性增强
提供多种操作选项：
- 只想查看内容 → 解密后直接关闭
- 需要复制内容 → 点击复制按钮
- 需要替换原文 → 点击替换按钮

### 改进点 3：反馈明确
每个操作都有清晰的视觉反馈和提示信息，用户知道当前状态和可执行的操作。

### 改进点 4：符合直觉
界面流程符合用户的心理预期：
1. 输入密码
2. 查看结果
3. 选择操作

## 技术实现

### 核心方法

```typescript
// 显示解密对话框（第一步：输入密码）
private async showDecryptDialog(encryptedText: string, savedRange: Range)

// 显示解密结果（第二步：展示结果和操作）
private showDecryptResult(dialog: HTMLElement, decryptedText: string, savedRange: Range, cleanup: () => void)
```

### 状态管理

对话框内容动态更新：
- 初始状态：密码输入界面
- 解密成功：结果展示界面
- 解密失败：显示错误提示，保持在输入界面

### 选区保存

使用 `Range.cloneRange()` 保存选区，确保在对话框操作后仍能正确替换文本。

## 国际化支持

新增国际化键：
- `decrypt`: 解密按钮文本
- `decrypting`: 解密中状态文本
- `enterPasswordToDecrypt`: 输入密码提示
- `decryptResultHint`: 解密结果提示
- `decryptedContent`: 解密内容标签
- `copyContent`: 复制按钮文本
- `copied`: 已复制反馈
- `replaceEncrypted`: 替换按钮文本
- `replaceSuccess`: 替换成功提示
- `replaceFailed`: 替换失败提示

## 使用示例

### 场景 1：只查看内容
1. 选中加密文本，右键解密
2. 输入密码，点击解密
3. 查看解密内容
4. 点击"关闭"按钮

### 场景 2：复制内容到其他地方
1. 选中加密文本，右键解密
2. 输入密码，点击解密
3. 点击"复制内容"按钮
4. 到其他地方粘贴使用

### 场景 3：替换加密文本
1. 选中加密文本，右键解密
2. 输入密码，点击解密
3. 确认内容无误
4. 点击"替换加密文本"按钮
5. 原文被替换为解密内容

## 对比总结

| 特性 | 优化前 | 优化后 |
|------|--------|--------|
| 解密流程 | 一步完成 | 两步确认 |
| 内容预览 | ❌ 无 | ✅ 有 |
| 操作选项 | 仅替换 | 复制/替换/关闭 |
| 误操作风险 | 较高 | 较低 |
| 用户控制 | 被动 | 主动 |
| 界面反馈 | 简单 | 丰富 |
| 符合直觉 | 一般 | 优秀 |
