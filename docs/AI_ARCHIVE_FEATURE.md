# AI问答封存功能说明

## 功能概述

AI内容生成器现在支持自动保存生成的内容到"AI问答封存"笔记本，实现智能分类和归档管理。

## 主要特性

### 1. 自动保存到封存笔记本

- 每次通过"生成"按钮生成内容后，会自动保存到"AI问答封存"笔记本
- 如果笔记本不存在，会自动创建
- 保存过程在后台进行，不影响用户继续操作

### 2. 智能内容分类

系统会根据生成内容的关键词自动分类到以下类别：

- **技术文档**: 包含代码、函数、API、接口、算法等技术相关内容
- **学习笔记**: 包含学习、笔记、总结、知识点等学习相关内容
- **创意想法**: 包含创意、想法、灵感、设计、方案等创意相关内容
- **问答记录**: 包含问题、回答、解答、疑问等问答相关内容
- **总结归纳**: 包含总结、归纳、概括、整理等归纳相关内容
- **其他**: 无法明确分类的内容

### 3. 自动创建子文档

- 每次生成的内容都会创建一个独立的子文档
- 文档标题自动从内容中提取（优先使用第一个标题）
- 文档名称格式：`标题-时间戳`，确保唯一性
- 子文档会自动归类到对应的分类文档下

### 4. 文档结构示例

```

├── 技术文档/
```

## 使用方式

### 自动保存（默认）

1. 在AI内容生成器中输入问题或需求
2. 点击"生成"按钮
3. 内容生成完成后，会自动保存到"AI问答封存"笔记本
4. 系统会显示保存成功的提示消息

### 手动保存

1. 生成内容后，点击结果区域的"保存"按钮
2. 内容会立即保存到"AI问答封存"笔记本
3. 可以多次保存同一内容（每次都会创建新的子文档）

## 技术实现

### 核心函数

1. **saveToArchiveNotebook()**: 主保存函数
2. **getOrCreateArchiveNotebook()**: 获取或创建封存笔记本
3. **autoClassifyContent()**: 智能内容分类
4. **getOrCreateCategoryDoc()**: 获取或创建分类文档
5. **generateDocTitle()**: 生成文档标题

### API调用

- `api.lsNotebooks()`: 列出所有笔记本
- `api.createNotebook()`: 创建新笔记本
- `api.sql()`: SQL查询分类文档
- `api.createDocWithMd()`: 创建Markdown文档

## 注意事项

1. 首次使用时会自动创建"AI问答封存"笔记本
2. 分类文档会在首次需要时自动创建
3. 每次生成都会创建新的子文档，不会覆盖已有内容
4. 文档标题从内容中自动提取，如果没有标题则使用"AI生成内容"
5. 时间戳精确到秒，确保文档名称唯一性

## 国际化支持

- `saveToArchive`: 保存到AI问答封存
- `savingToArchive`: 正在保存到AI问答封存...
- `saveToArchiveSuccess`: 已保存到AI问答封存
- `saveToArchiveFailed`: 保存到AI问答封存失败
