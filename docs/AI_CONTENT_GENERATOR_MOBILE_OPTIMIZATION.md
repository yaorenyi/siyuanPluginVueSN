# AI内容生成器移动端优化

## 概述

针对AI内容生成器功能进行移动端适配优化，提升移动设备上的用户体验，特别是针对问答场景的优化。

## 优化内容

### 1. 移动端检测

**实现方式：**
- 检测思源笔记的移动端环境（`body--mobile` class）
- 检测屏幕宽度（≤768px）
- 响应窗口大小变化

**代码位置：**
```typescript
// AIContentGeneratorPanel.vue
const isMobile = ref(false);

const checkIsMobile = () => {
  const isSiyuanMobile = document.body.classList.contains('body--mobile');
  const isSmallScreen = window.innerWidth <= 768;
  isMobile.value = isSiyuanMobile || isSmallScreen;
};
```

### 2. 移动端功能调整

#### 2.1 隐藏封存按钮
- **位置：** 结果操作栏
- **原因：** 移动端屏幕空间有限，减少不常用功能
- **实现：** `v-if="!editMode && !isMobile"`

#### 2.2 禁用自动封存
- **位置：** 生成成功后的自动保存逻辑
- **原因：** 移动端用户可能不需要自动保存，减少不必要的操作
- **实现：** `if (!editMode.value && !isMobile.value)`

#### 2.3 自动收起面板 ⭐ 新增
- **触发时机：** 
  - 点击"生成"按钮时
  - 使用AI编辑工具时（润色、扩写、精简等）
  - 点击"智能分析"时
  - 应用AI建议时
- **收起内容：**
  - 对话设置面板
  - 提示词选择器
- **原因：** 让用户立即看到生成的内容，无需手动滚动
- **实现：**
  ```typescript
  if (isMobile.value) {
    showSettings.value = false;
    showPromptSelector.value = false;
  }
  ```

### 3. UI优化

#### 3.1 触摸友好的交互
- **按钮尺寸：** 最小 44x44px（符合移动端触摸标准）
- **间距增加：** 按钮间距从 6px 增加到 8-10px
- **字体放大：** 基础字体从 12-13px 增加到 14-15px

#### 3.2 布局优化
- **头部工具栏：** 
  - 支持换行显示
  - 标题和操作按钮分行
  - 粘性定位，滚动时保持可见

- **输入区域：**
  - 粘性定位在底部
  - 增加阴影提升层次感
  - 输入框最小高度 100px

- **结果操作栏：**
  - 垂直布局，按钮占满宽度
  - 重要按钮（应用、插入、停止）独占一行
  - 其他按钮两列布局

#### 3.3 提示词选择器优化
- 最大高度限制为 60vh
- 列表项增大触摸区域（padding: 14px）
- 添加 `:active` 状态反馈

#### 3.4 AI编辑工具栏优化
- 按钮两列布局（flex: 1 1 calc(50% - 4px)）
- "智能分析"按钮独占一行
- 按钮高度增加到 44px

#### 3.5 Markdown预览优化
- 字体大小增加到 15px
- 行高增加到 1.7
- 表格支持横向滚动（-webkit-overflow-scrolling: touch）
- 代码块支持触摸滚动

### 4. 响应式断点

```scss
@media (max-width: 768px), (hover: none) {
  // 移动端样式
}

.body--mobile {
  // 思源笔记移动端特定样式
}
```

### 5. 滚动优化

- **平滑滚动：** `-webkit-overflow-scrolling: touch`
- **滚动边界：** `overscroll-behavior: contain`
- **防止过度滚动**

## 技术细节

### 移动端检测逻辑

1. **思源环境检测：** 检查 `body--mobile` class
2. **屏幕宽度检测：** `window.innerWidth <= 768`
3. **动态响应：** 监听 `resize` 事件
4. **清理机制：** 组件卸载时移除事件监听

### 样式优化策略

1. **触摸目标：** 最小 44x44px（Apple HIG 标准）
2. **字体缩放：** 基础字体 14-15px（移动端可读性）
3. **间距调整：** 增加 20-30% 的间距
4. **布局转换：** 横向布局转为纵向堆叠

## 用户体验改进

### 问答场景优化

1. **快速输入：** 大尺寸输入框和按钮
2. **清晰阅读：** 增大字体和行高
3. **便捷操作：** 大触摸区域，减少误触
4. **流畅滚动：** 优化滚动性能
5. **自动收起：** 生成时自动收起设置面板，直接显示结果 ⭐

### 空间利用

1. **隐藏非必要功能：** 封存按钮在移动端隐藏
2. **垂直布局：** 充分利用纵向空间
3. **粘性定位：** 重要区域保持可见
