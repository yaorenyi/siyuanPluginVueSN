# AI 内容生成器 - 自动加载文档功能

## 功能概述

AI 内容生成器现在支持**自动获取当前激活文档**，无需手动点击"选择文档"按钮，大幅提升使用体验。

---

## 核心特性

### 🔄 自动加载
- 监听思源笔记的文档切换
- 自动获取当前激活窗口的文档
- 智能防抖（500ms），避免频繁切换

### 🎯 智能判断
- 检测文档变化，避免重复加载
- AI 生成中不自动切换
- 手动选择文档时不自动覆盖

### ⚡ 开关控制
- 随时切换自动/手动模式
- 视觉状态反馈（动画徽章）

---

## 使用指南

### 自动模式（默认）

1. 打开 AI 内容生成器面板
2. 切换到任意文档
3. 面板自动加载当前文档
4. 文档名称旁显示 **"自动"** 徽章

```
┌─────────────────────────────────────┐
│ 📄 我的文档 [自动]     [🔄] [✕]    │
└─────────────────────────────────────┘
    └─ 自动加载徽章    开关  清除
```

### 手动模式

1. 点击开关按钮（🔄 图标）
2. 按钮变为灰色（暂停状态）
3. 手动点击"选择文档"按钮

```
┌─────────────────────────────────────┐
│ 📄 选择文档              [⏸]       │
└─────────────────────────────────────┘
                          └─ 已暂停
```

### 开关按钮说明

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| 开启 | 🔄 | 主题色 | 自动加载激活，有脉冲动画 |
| 关闭 | ⏸ | 灰色 | 手动模式，不自动加载 |

## 工作原理

### 监听机制

```typescript
// 1. 监听窗口激活事件
document.addEventListener('click', autoLoadCurrentDoc)

// 2. 监听 DOM 变化（标签页切换）
const observer = new MutationObserver(() => {
  autoLoadCurrentDoc()
})

// 3. 获取激活窗口文档 ID
const docId = document.querySelector('.layout__wnd--active .protyle')
  ?.querySelector('.protyle-background')
  ?.getAttribute('data-node-id')
```

### 防抖机制

```
用户操作 → 等待 500ms → 检查文档是否变化 → 加载文档
```

避免快速切换文档时频繁加载。

### 智能判断逻辑

```typescript
// ✅ 允许自动加载
if (新文档 && !正在生成 && !手动选择了其他文档) {
  加载文档()
}

// ❌ 阻止自动加载
if (文档相同 || 正在生成 || 手动选择了其他文档) {
  跳过()
}
```

## 常见问题

### Q: 自动加载会影响性能吗？
**A:** 不会。使用防抖机制（500ms），避免频繁加载，仅在文档真正切换时才加载。

### Q: 如何停止自动加载？
**A:** 点击文档选择器右侧的开关按钮即可。

### Q: 自动加载会覆盖我手动选择的文档吗？
**A:** 不会。如果手动选择了文档，自动加载会跳过，不会覆盖。

### Q: 切换文档时为什么要等待？
**A:** 这是为了避免快速切换时频繁加载，500ms 防抖确保只在文档稳定后才加载。
